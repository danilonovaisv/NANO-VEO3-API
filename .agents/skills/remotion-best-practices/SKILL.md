# 🎬 Remotion — Programmatic Video Creation Skill

## Overview

Remotion enables video creation using React components. This skill defines architecture patterns, best practices, and common recipes for the VIDEOS-VEO3 studio.

## When to Use

- Creating videos with dynamic data (charts, text overlays, counters)
- Building template-based video series (intros, outros, episode cards)
- Composing multiple AI-generated clips with transitions and overlays
- Rendering videos that require precise frame-level timing control

## Architecture Principles

### 1. Modular Composition Structure

```tsx
// Root.tsx — Register all compositions
import { Composition } from 'remotion';
import { CyberpunkIntro } from './scenes/CyberpunkIntro';
import { DataViz } from './scenes/DataViz';
import { Outro } from './scenes/Outro';

export const RemotionRoot: React.FC = () => {
  return (
    <>
      <Composition
        id="CyberpunkIntro"
        component={CyberpunkIntro}
        durationInFrames={300} // 10 seconds at 30fps
        fps={30}
        width={3840}
        height={2160}
      />
      <Composition
        id="FullVideo"
        component={FullVideo}
        durationInFrames={900} // 30 seconds
        fps={30}
        width={3840}
        height={2160}
      />
    </>
  );
};
```

### 2. Scene Isolation with `<Sequence>`

Each scene must be an independent `<Sequence>` block:

```tsx
import { Sequence, useCurrentFrame, useVideoConfig } from 'remotion';

export const FullVideo: React.FC = () => {
  return (
    <AbsoluteFill>
      <Sequence from={0} durationInFrames={150}>
        <CyberpunkIntro />
      </Sequence>
      <Sequence from={150} durationInFrames={300}>
        <DataViz data={chartData} />
      </Sequence>
      <Sequence from={450} durationInFrames={150}>
        <Outro />
      </Sequence>
    </AbsoluteFill>
  );
};
```

### 3. Animation Timing Rules

```tsx
// ✅ CORRECT — Time-based with interpolation
import { interpolate, spring } from 'remotion';

const frame = useCurrentFrame();
const { fps } = useVideoConfig();

const opacity = interpolate(frame, [0, 30], [0, 1], {
  extrapolateRight: 'clamp',
});

const scale = spring({
  frame,
  fps,
  config: { damping: 100, stiffness: 200, mass: 0.5 },
});
```

### 4. Audio Synchronization

```tsx
import { Audio, Sequence } from 'remotion';

// Sync audio with visual sequences in independent blocks
<Sequence from={0} durationInFrames={900}>
  <Audio src={staticFile('audio/bg_music.mp3')} volume={0.6} />
</Sequence>
<Sequence from={0} durationInFrames={150}>
  <Audio src={staticFile('audio/sfx_whoosh.mp3')} volume={1.0} />
</Sequence>
```

## Rendering Commands

```bash
# Preview in browser
npx remotion preview src/Root.tsx

# Render single composition
npx remotion render src/Root.tsx CyberpunkIntro public/out/cyberpunk_intro.mp4

# Render with custom codec settings
npx remotion render src/Root.tsx FullVideo public/out/full_video.mp4 \
  --codec h264 \
  --crf 18 \
  --frames 0-899
```

## Anti-Patterns ❌

- **DON'T** use `setTimeout` or `setInterval` — use `useCurrentFrame()` instead
- **DON'T** manipulate DOM directly — Remotion renders frames sequentially
- **DON'T** use state for animation progress — derive from frame number
- **DON'T** hardcode FPS values — always use `useVideoConfig()`
- **DON'T** import assets with relative paths — use `staticFile()` helper
